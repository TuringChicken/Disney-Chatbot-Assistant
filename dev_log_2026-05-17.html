核心实现：自动分类路由

路由算法

对每个分类计算查询中命中的特征关键词数量，取最高分分类；若全部为 0 则使用全量索引。时间复杂度 O(C×K)，C=5 分类，K≤15 关键词，查询在毫秒内完成。

_ROUTE_RULES = [
    ("特殊情况与应急预案", ["紧急", "走失", "急救", "台风", "闭园", "emergency", ...]),
    ("内部知识与工具",    ["CRM", "工单", "员工", "P1", "P2", "Salesforce", ...]),
    ("客户关系与支持话术", ["话术", "奇迹时刻", "补偿", "HEARD", "magic moment", ...]),
    ("运营流程与标准作业程序", ["退款", "退票", "改期", "预订", "客诉", "refund", ...]),
    ("产品与服务详情",   ["门票", "年票", "餐厅", "身高", "价格", "ticket", ...]),
]

def auto_detect_category(query: str) -> str:
    q = query.lower()
    scores = {cat: sum(1 for kw in kws if kw.lower() in q)
              for cat, kws in _ROUTE_RULES}
    best_cat, best_score = max(scores.items(), key=lambda x: x[1])
    detected = best_cat if best_score > 0 else "全部"
    print(f"[路由] 查询='{query[:30]}' → 分类='{detected}' (得分={best_score})")
    return detected

懒加载索引缓存

全量索引在应用启动时预加载进 _index_cache，分类索引在首次被路由命中时才加载，避免启动时占用过多内存。

_index_cache: dict = {}

def get_index(category: str):
    if category not in _index_cache:
        idx_file, meta_file = CATEGORY_INDEXES[category]
        _index_cache[category] = load_index(idx_file, meta_file)
    return _index_cache[category]

已知限制 / 后续优化方向

  • 路由为关键词匹配：纯词频策略对同义词或跨类模糊查询可能路由不准；可改为向量分类模型。
  • 香港 PDF 语义覆盖过宽：cat1_products 中该文档占比高，部分上海特定问题仍受干扰；可在 ingestion 阶段为不同城市文档打标签，向量过滤时加 metadata filter。
  • 媒体资源数量有限：当前视频仅一条，图片数量较少，媒体检索体验受限。
  • Embedding API 延迟：每次查询调用 DashScope 远程 Embedding，P99 延迟约 1–2 秒；可在本地缓存常见查询向量。
  • Chunk 大小未做 Token 限制：当前分块策略未限制每个 chunk 的最大 token 数，导致部分 chunk 语义覆盖过宽，干扰向量相似度排序。建议在 ingestion 阶段按固定 token 上限（如 256–512 tokens）切分，确保每个 chunk 语义聚焦。
  • 图片召回测试全部失败：当前图片检索逻辑依赖查询中出现图片意图关键词（如"海报"、"图片"）才触发，且距离阈值 MEDIA_DISTANCE_THRESHOLD=3.0 过于宽松，实际召回率低。需重新审查图片向量化方式（是否应将图片描述文本一同嵌入）以及距离阈值的合理范围。
