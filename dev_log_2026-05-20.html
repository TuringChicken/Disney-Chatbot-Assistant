核心实现：RRF 融合与 Qwen3-VL 重排序

RRF 融合算法

将三路召回结果合并，同一文档出现在多路时累加得分，天然实现跨通道去重与提权。

def _rrf_merge(channels: list, top_k: int = RRF_TOP_K) -> list:
    doc_rrf: dict = {}
    doc_data: dict = {}
    for ch in channels:
        for rank, r in enumerate(ch):
            did = r["doc_id"]
            doc_rrf[did] = doc_rrf.get(did, 0.0) + 1.0 / (RRF_K + rank)
            if did not in doc_data:
                doc_data[did] = r
    ranked = sorted(doc_rrf.items(), key=lambda x: -x[1])
    result = []
    for did, score in ranked[:top_k]:
        entry = dict(doc_data[did])
        entry["rrf_score"] = round(score, 6)
        result.append(entry)
    return result

Qwen3-VL 并发重排序

对 RRF top-15 候选集并发调用 VL 模型打分，文本和图片走不同评分路径，最终按分数降序取 top-5。

def _rerank_with_vl(query, candidates, top_k=RERANK_TOP_K):
    scores = {}
    with ThreadPoolExecutor(max_workers=min(len(candidates), 4)) as pool:
        futures = {pool.submit(_score_relevance, query, r): r["doc_id"]
                   for r in candidates}
        for future in as_completed(futures):
            did = futures[future]
            scores[did] = future.result()   # 0–10 float

    ranked = sorted(candidates, key=lambda r: -scores.get(r["doc_id"], 0))
    for r in ranked:
        r["rerank_score"] = round(scores.get(r["doc_id"], 0), 2)
    return ranked[:top_k]


def _score_image(query, image_path) -> float:
    with open(image_path, "rb") as f:
        b64 = base64.b64encode(f.read()).decode("utf-8")
    mime = "jpeg" if image_path.lower().endswith((".jpg", ".jpeg")) else "png"
    resp = client.chat.completions.create(
        model=VL_RERANK_MODEL,
        messages=[{"role": "user", "content": [
            {"type": "image_url",
             "image_url": {"url": f"data:image/{mime};base64,{b64}"}},
            {"type": "text",
             "text": f'请对该图片与问题的相关性打分（0-10整数）...\n问题：{query}\n仅返回JSON：{{"score": <分数>}}'},
        ]}]
    )
    return float(json.loads(resp.choices[0].message.content)["score"])

问题分析与修复

FIX 图片召回缺失 — 根本原因与解决方案

问题：询问"迪士尼邮轮价格表"时，检索来源中没有任何图片文件，9 张邮轮价格图片完全缺失。

根本原因：旧架构通过关键词（"海报"、"图片"、"图"）触发图片意图检测，用户自然语言提问不包含这些词时，图片通道根本不参与检索。即使参与，也与文本向量混在同一通道，图片向量天然语义稀疏，被文本向量压制。

解决方案：为图片建立独立的向量召回通道（_vector_channels 拆分返回 text_ch 和 image_ch），图片在自己通道内参与 top-k 竞争，无需关键词触发；BM25 通道对图片元数据的 content 字段（"[图片] 迪士尼邮轮价格1.jpeg"）同样打分，进一步提升图片召回率。

FIX test_irrelevant_query 测试用例设计失败

问题：test_irrelevant_query_scores_zero：噪声查询 "xyzzy_nonsense_word_123" 因其中含有 "word" 子串，而 "word" 在小语料库中 IDF 极高，导致 BM25 得分非零，测试断言 score == 0 失败。

修复：废弃绝对零分断言，改为相对顺序测试——"邮轮价格" 查询在邮轮图片上的 BM25 得分应高于无关查询，改用 assertGreater(relevant_score, irrelevant_score)。

已知限制 / 后续优化方向

  • 重排序延迟较高：对 15 个候选并发调用 Qwen3-VL，在 4 并发下耗时约 3–6 秒；可将 VL_RERANK_MODEL 降级为更轻量模型（如 qwen-vl-plus）以换取速度。
  • 图片 BM25 得分依赖文件名：BM25 对图片的打分基于 "[图片] 迪士尼邮轮价格X.jpeg" 这样的内容字段，实质上是文件名匹配；若图片文件名语义不清则无效，后续可考虑在 ingestion 阶段为图片生成文字描述。
  • Chunk 大小未按 Token 限制：当前分块按字符数（500 字）切分，不同模型的 token 计数差异可能导致部分 chunk 过长，影响向量语义聚焦度。
  • 路由关键词为静态配置：当用户使用同义词或跨类模糊查询时，路由可能错误命中分类索引；可引入向量分类器替代词频路由。
  • 缺乏端到端测试：现有测试覆盖召回阶段，重排序和生成阶段尚无自动化测试；建议补充 golden-set 评估（固定查询集对比实际回答）。
